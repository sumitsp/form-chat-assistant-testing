Architecture at a Glance

MySQL (Structured Eligibility Gates)

  • Binary pass/fail rules: FICO min, LTV, loan amount, DSCR floor
  • Program-level flags: FTHB, IO, entity vesting, DTI cap, etc.
  • LTV matrix rows by FICO band x loan amt x occupancy x doc type
  • Documentation requirements output (step-2 answer)
  • Geographic restrictions (state/county/city level)
  • Credit event tiered seasoning rules

Qdrant (Semantic RAG — Guideline Text)

  • Complex conditional rules requiring natural language explanation
  • Calculation methodologies (DSCR, asset utilization, residual income)
  • Nuanced underwriting guidance (employment gaps, self-employment, etc.)
  • Borrower eligibility prose (citizenship types, co-borrower rules)
  • Transaction type rules (cash-out, delayed financing, construction)
  • Property eligibility details (condo types, rural, leasehold)

2. programs

CREATE TABLE programs (
  id                    SMALLINT UNSIGNED  AUTO_INCREMENT PRIMARY KEY,
  lender_id             TINYINT UNSIGNED   NOT NULL,
  program_code          VARCHAR(60)        NOT NULL UNIQUE, -- e.g. 'NQM_FLEX_SUPREME'
  program_name          VARCHAR(120)       NOT NULL,

  -- Loan structure
  is_second_lien        TINYINT(1)         NOT NULL DEFAULT 0,
  is_heloc              TINYINT(1)         NOT NULL DEFAULT 0,
  is_dscr_program       TINYINT(1)         NOT NULL DEFAULT 0,
  is_multifamily        TINYINT(1)         NOT NULL DEFAULT 0,
  is_cross_collateral   TINYINT(1)         NOT NULL DEFAULT 0,
  is_foreign_national   TINYINT(1)         NOT NULL DEFAULT 0,
  is_itin_program       TINYINT(1)         NOT NULL DEFAULT 0,
  min_units             TINYINT UNSIGNED   DEFAULT 1,
  max_units             TINYINT UNSIGNED   DEFAULT 4,

  -- Loan amounts
  loan_amt_min          INT UNSIGNED       NOT NULL DEFAULT 75000,
  loan_amt_max          INT UNSIGNED       NOT NULL,
  loan_amt_max_secondary INT UNSIGNED      NULL,    -- e.g. FN sub-limit within program

  -- FICO
  fico_min              SMALLINT UNSIGNED  NOT NULL,
  fico_max              SMALLINT UNSIGNED  NULL,    -- NULL = no stated cap
  no_fico_allowed       TINYINT(1)         DEFAULT 0, -- FN programs with no-score path

  -- DTI / Residual income
  max_dti               DECIMAL(4,1)        NULL,    -- NULL for pure DSCR programs
  residual_income_required TINYINT(1)      DEFAULT 0,
  residual_income_base  DECIMAL(8,2)        NULL,    -- e.g. 2500.00 (DH Expanded Prime)
  residual_per_dependent DECIMAL(6,2)       NULL,    -- 150.00 (DH Non-Prime)

  -- DSCR
  dscr_min              DECIMAL(4,2)        NULL,    -- e.g. 1.00, 0.75, 1.15
  dscr_min_str          DECIMAL(4,2)        NULL,    -- STR-specific DSCR floor
  dscr_allows_below_one TINYINT(1)         DEFAULT 0,
  dscr_calc_method      ENUM('gross_rents_pitia','gross_rents_itia','noi_nds','n/a')
                                             DEFAULT 'n/a',

  -- IO
  io_eligible           TINYINT(1)         DEFAULT 0,
  io_min_fico           SMALLINT UNSIGNED  NULL,
  io_max_ltv            DECIMAL(4,1)        NULL,
  io_fthb_eligible      TINYINT(1)         DEFAULT 0,
  io_occupancy_restrictions VARCHAR(200)   NULL,

  -- Occupancy / Property
  occupancy_types       JSON               NOT NULL,
  -- ["primary","second","investment"]
  property_types_allowed   JSON            NULL,
  property_types_ineligible JSON           NULL,
  max_acreage           DECIMAL(5,1)        NULL,    -- 5.0, 20.0
  mixed_use_max_commercial_pct DECIMAL(4,1) NULL,  -- 49.99 (Verus mixed use)

  -- Transaction types
  loan_purposes_allowed JSON               NOT NULL,
  -- ["purchase","rate_term","cash_out"]
  product_types         JSON               NOT NULL,
  -- ["30yr_fixed","5_6_arm","15yr_fixed","io_30yr"]
  prepayment_penalty_eligible TINYINT(1)  DEFAULT 0,
  buydown_2_1_ok        TINYINT(1)         DEFAULT 0,

  -- Cash-out limits
  cash_out_max_ltv_threshold   INT UNSIGNED NULL,
  cash_out_max_amt_below       INT UNSIGNED NULL,
  cash_out_max_amt_above       INT UNSIGNED NULL,
  cash_in_hand_max_65plus_ltv  INT UNSIGNED NULL,  -- Verus DSCR Plus: $500K if >=65%
  cash_in_hand_max_below_65    INT UNSIGNED NULL,  -- $1M if <65%

  -- Borrower eligibility flags
  us_citizen_ok         TINYINT(1)         DEFAULT 1,
  perm_resident_ok      TINYINT(1)         DEFAULT 1,
  non_perm_resident_ok  TINYINT(1)         DEFAULT 0,
  itin_ok               TINYINT(1)         DEFAULT 0,
  daca_ok               TINYINT(1)         DEFAULT 0,
  foreign_national_ok   TINYINT(1)         DEFAULT 0,
  non_occ_coborrower_ok TINYINT(1)         DEFAULT 0,
  entity_vesting_ok     TINYINT(1)         DEFAULT 0,
  entity_layering_prohibited TINYINT(1)   DEFAULT 0,
  domestic_llc_required TINYINT(1)         DEFAULT 0,
  personal_guarantor_required TINYINT(1)  DEFAULT 0,
  subordinate_financing_allowed TINYINT(1) DEFAULT 0,
  escrow_waiver_allowed TINYINT(1)         DEFAULT 1,
  ach_required          TINYINT(1)         DEFAULT 0,  -- NQM Foreign National

  -- FTHB / First Time Investor
  fthb_eligible         TINYINT(1)         DEFAULT 0,
  fthb_max_loan         INT UNSIGNED       NULL,    -- e.g. 1500000 (NQM Flex Supreme)

  bk_seasoning_months          SMALLINT UNSIGNED NULL,

  -- Housing history

  housing_history_code         VARCHAR(30)       NULL,   -- '0x30x12','1x30x12','0x30x24'
  housing_history_1x30_ltv_reduction DECIMAL(4,1) NULL, -- e.g. 5.0 (NQM Investor DSCR)

  pl_requires_bank_stmt TINYINT(1)         DEFAULT 0, -- DH Non-Prime: P&L needs 2mo BS
  gift_funds_min_borrower_pct DECIMAL(4,1) NULL,    -- DH ITIN: 5%

  -- STR (Short-Term Rental)
  str_eligible          TINYINT(1)         DEFAULT 0,
  str_min_dscr          DECIMAL(4,2)        NULL,
  str_min_fico          SMALLINT UNSIGNED  NULL,
  str_ltv_reduction     DECIMAL(4,1)        NULL,    -- e.g. 5.0%
  str_refi_only         TINYINT(1)         DEFAULT 0,

  -- Seller concessions
  seller_concessions_max_pct DECIMAL(4,1)  NULL,    -- 3.0 (DH DSCR 5-9 Unit)

  -- Ownership / seasoning
  min_ownership_seasoning_months TINYINT UNSIGNED NULL,
  asset_seasoning_months         TINYINT UNSIGNED NULL,

  -- Appraisal
  appraisals_required   TINYINT UNSIGNED   DEFAULT 1, -- NQM Super Jumbo: 2

  -- AUS / Sub-products
  aus_allowed           TINYINT(1)         DEFAULT 0,
  express_du_available  TINYINT(1)         DEFAULT 0, -- NQM Flex Select

  -- Cross-collateral specific
  xc_min_properties     TINYINT UNSIGNED   NULL,    -- 2
  xc_max_properties     TINYINT UNSIGNED   NULL,    -- 25
  xc_min_total_loan     INT UNSIGNED       NULL,    -- $200K
  xc_max_per_property   INT UNSIGNED       NULL,    -- $1.5M
  xc_min_property_value INT UNSIGNED       NULL,    -- $100K
  xc_property_dscr_min  DECIMAL(4,2)        NULL,    -- .75 property-level DSCR

  -- Misc
  exceptions_allowed    TINYINT(1)         DEFAULT 1,
  notes                 TEXT               NULL,

  -- Metadata
  effective_date        DATE               NOT NULL,
  superseded_date       DATE               NULL,
  is_active             TINYINT(1)         DEFAULT 1,
  created_at            TIMESTAMP          DEFAULT CURRENT_TIMESTAMP,
  updated_at            TIMESTAMP          DEFAULT CURRENT_TIMESTAMP
                                             ON UPDATE CURRENT_TIMESTAMP,

  FOREIGN KEY (lender_id) REFERENCES lenders(id),
  INDEX idx_lender  (lender_id),
  INDEX idx_active  (is_active),
  INDEX idx_dscr    (is_dscr_program),
  INDEX idx_second  (is_second_lien)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

3. ltv_matrix

Each row represents one cell in a program LTV grid. The chatbot queries this to find the maximum LTV given all borrower inputs. 

CREATE TABLE ltv_matrix (
  id              INT UNSIGNED      AUTO_INCREMENT PRIMARY KEY,
  program_id      SMALLINT UNSIGNED NOT NULL,

  -- Matching dimensions (all used together to find the best matching row)
  fico_min        SMALLINT UNSIGNED NOT NULL,
  fico_max        SMALLINT UNSIGNED NULL,     -- NULL = 850
  loan_amt_min    INT UNSIGNED      NOT NULL DEFAULT 0,
  loan_amt_max    INT UNSIGNED      NULL,     -- NULL = no cap
  occupancy       ENUM('primary','second','investment','any')
                                    NOT NULL DEFAULT 'any',
  units           ENUM('1','2','3','4','5-8','5-9','1-4','any')
                                    NOT NULL DEFAULT 'any',
  doc_type        ENUM('full_doc','bank_stmt_12','bank_stmt_24','pl_only',
                       'pl_2mo_bs','wvoe','asset_util','1099',
                       'dscr_rental','itin','non_traditional','any')
                                    NOT NULL DEFAULT 'any',
  loan_purpose    ENUM('purchase','rate_term','cash_out','any')
                                    NOT NULL DEFAULT 'any',
  dscr_band       ENUM('gte_1_00','gte_0_75_lt_1_00','lt_0_75','any')
                                    DEFAULT 'any',
  is_io           TINYINT(1)        NOT NULL DEFAULT 0,
  is_str          TINYINT(1)        NOT NULL DEFAULT 0,
  is_fthb         TINYINT(1)        DEFAULT 0,
  state_override  CHAR(2)           NULL,   -- NULL=all states; 'CT','FL',etc.

  -- Output
  max_ltv         DECIMAL(4,1)      NOT NULL,
  max_cltv        DECIMAL(4,1)      NULL,   -- for second liens and HELOCs
  ltv_note        VARCHAR(300)      NULL,   -- e.g. '5% reduction for loan <$150K'
  sort_order      SMALLINT          DEFAULT 0, -- lower = more specific = higher priority

  FOREIGN KEY (program_id) REFERENCES programs(id),
  INDEX idx_prog_fico (program_id, fico_min, fico_max),
  INDEX idx_prog_occ  (program_id, occupancy),
  INDEX idx_prog_doc  (program_id, doc_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

4. credit_event_tiers NEW

For programs with tiered LTV caps based on credit event seasoning months. Examples: NQM Flex Select (3 tiers), Verus Prime Ascent (3 tiers). 

CREATE TABLE credit_event_tiers (
  id                INT UNSIGNED      AUTO_INCREMENT PRIMARY KEY,
  program_id        SMALLINT UNSIGNED NOT NULL,
  min_months        SMALLINT UNSIGNED NOT NULL,  -- >= this many months seasoning
  max_months        SMALLINT UNSIGNED NULL,       -- NULL = no upper bound (best tier)
  max_ltv_purchase  DECIMAL(4,1)      NULL,       -- NULL = use base ltv_matrix
  max_ltv_refi      DECIMAL(4,1)      NULL,
  max_loan_amount   INT UNSIGNED      NULL,       -- NULL = no restriction
  ltv_reduction_pct DECIMAL(4,1)      NULL,       -- alternative: delta from base
  tier_note         VARCHAR(300)      NULL,
  FOREIGN KEY (program_id) REFERENCES programs(id),
  INDEX idx_prog_months (program_id, min_months)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- NQM Flex Select (program_code='NQM_FLEX_SELECT')
-- Tier 1: >=48 months: use base matrix (no LTV cap)
-- Tier 2: >=36 months, <48 months: max 80% LTV
-- Tier 3: >=24 months, <36 months: max 70% LTV

-- Verus Prime Ascent (program_code='VERUS_PRIME_ASCENT')
-- Tier 1: >=36 months: use base matrix
-- Tier 2: >=24 months, <36 months: max 80% purchase / 75% refi / $1.5M max loan
-- Tier 3: >=12 months, <24 months: max 70% / $1M max loan

ETL Processing Notes

Matrix PDF → MySQL Extraction

LTV Matrix Parsing Strategy

Each program matrix is a 2D grid where rows = FICO bands / loan amount ranges and columns = occupancy types x doc types. The parser must:

  1. Identify the header row to determine column → (occupancy, doc_type) mapping
  2. Parse each data row into (fico_min, fico_max, loan_amt_min, loan_amt_max) from the row label
  3. Extract the LTV value for each column intersection → one ltv_matrix row per cell
  4. Detect and handle LTV reductions noted in footnotes (IO, STR, state overlays)
  5. Insert separate rows for state-overlay LTV caps (e.g., Verus CT/FL/IL/NJ/NY overlays)
  6. Flag programs with tiered credit event rules → insert rows into credit_event_tiers

Program-Level Flag Extraction

Matrix "General Requirements" and "Borrower Eligibility" sections at the bottom of each program page contain program-level flags. Parse these as key-value pairs into the programs table. Items that cannot be reduced to a boolean or number go to Qdrant instead.

Verus Prime Ascent has tiered credit event LTV reductions embedded inside the matrix cells. These produce credit_event_tiers rows rather than separate ltv_matrix rows.

Guideline PDF → Qdrant Chunking

Chunking Rules

  • Target size: 400–900 tokens per chunk. Preserve complete paragraphs.
  • Chunk boundaries: Always break at section headings. Never mid-sentence.
  • Overlap: 50-token sliding overlap between adjacent chunks within a section.
  • Heading inheritance: Each chunk carries full breadcrumb heading_path.
  • Table handling: Convert inline tables to prose or markdown before embedding. Do not embed raw PDF coordinates.
  • Program specificity: If section header names a specific program, set program_ids to that program only. If section is general, set program_ids to empty array (applies to all lender programs).

Chatbot Conversation Flow

Phase 1 — Eligibility Screening (MySQL)

1
Loan purpose — purchase / rate-term refi / cash-out refi
2
Property state → immediately query geographic_restrictions to eliminate ineligible programs
3
Occupancy — primary / second home / investment
4
Property type + units — SFR / condo / 2-4 unit / 5-8 unit / mixed use
5
Loan amount → filter by programs.loan_amt_min and programs.loan_amt_max
6
FICO score → filter by programs.fico_min and programs.fico_max
7
DSCR? If investment property: ask for rents vs PITI or known DSCR value → filter DSCR programs by dscr_min and dscr_allows_below_one
8
Doc type — full doc / bank stmt / P&L / WVOE / asset util → filter by programs.doc_types_allowed
9
Borrower type — US citizen / PRA / ITIN / FN / DACA → filter by citizenship flags
10
Credit event? BK / FC / SS / DIL + months ago → check programs.credit_event_seasoning_months and credit_event_tiers
11
Housing lates? → check programs.housing_history_code → compute LTV reduction if applicable
12
FTHB? / First time investor? → apply flag filters and loan cap
13
IO requested? → filter by io_eligible / io_min_fico / io_max_ltv
14
Entity vesting? / Lender preference? → final filters
15
Query ltv_matrix for max LTV given all inputs → present matching programs with max LTV to borrower

Phase 2 — Documentation Checklist (MySQL doc_requirements)

Once the program is selected, query doc_requirements for the borrower's (doc_type, borrower_type) combination. Return the JSON document checklist as a formatted list.

Phase 3 — Guideline Q&A (Qdrant RAG)

Borrower or LO has follow-up questions. Use matched program(s) from Phase 1 to scope the Qdrant search:

  • Filter by lender_id of the winning program
  • Filter by program_ids containing the winning program code (or empty = all-lender chunks)
  • Always filter is_superseded = false
  • Classify question topic → add section_type filter
  • Return top-K chunks → pass to LLM with system prompt for answer synthesis
  • Include source_file + page_number in response for LO citation reference
